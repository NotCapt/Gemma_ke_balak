The Pipeline That Thinks

Each stage earns the right to run only if the previous one found something worth escalating. Cheap filters first, expensive reasoning only when justified.

Stage 1 — Video Anomaly Detection

A lightweight CLIP-TSA model watches every camera stream continuously. It's cheap enough to run 24/7 on all feeds, and its only job is to filter out the ~90% of footage where nothing's happening.

⚡
Model: CLIP-TSA (pretrained on UCF-Crime)
Anomaly score 0–1 per rolling clip window
🔥
Why it matters: Kills noise before anything expensive runs.
~50–200ms per clip latency

Stage 2 — Semantic Confirmation

Only clips crossing the VAD threshold reach Gemma. It confirms or rejects the anomaly, gives a natural-language reasoning description, and flags whether a person is involved. This is the "reasoning layer" competitors lack.

🧠
Model: Gemma (on-device)
Frame collage input — 5 frames stitched into single image
🔒
Privacy: No raw footage ever leaves the edge device.
On-device processing resolves cloud latency/privacy risk

Stage 3 — Person Localization

Runs only if Gemma flags person_involved: yes. Non-person anomalies (fire, object left behind) skip straight to logging — no wasted compute. YOLOv8n provides fast, accurate person detection with bounding boxes.

🎯
Model: YOLOv8n (nano variant, COCO-pretrained)
~10–20ms per frame (GPU), ~90%+ person recall
⚙️
Gated compute: Never runs on non-person anomalies.
Saves compute by only detecting when semantically justified

Stage 4 & 5 — Re-ID + Cross-Camera Matching

YOLO crop → OSNet embedding (512-dim). Embeddings are matched across cameras using cosine similarity. Benchmark ~94% rank-1 accuracy, realistic ~75–85% under field conditions. Stated honestly, not oversold.

🔗
Model: OSNet Re-ID (2.2M params, ~5–10ms/crop)
Cosine similarity threshold ~0.7 for cross-camera matching
📹
Cross-camera: Same person flagged across multiple feeds.
In-memory embedding store with camera_id + timestamp

Stage 6 & 7 — Forensic Log + Alert Dashboard

Every stage's output is written to an append-only, hash-chained JSON log. By the time something reaches a human operator, there's a full trail behind it — not just a flashing red alert with no context.

🔗
Hash-chained: Each entry references the previous hash.
Tamper-evident, append-only log for forensic-grade verification
👤
Human-in-the-loop: Operator can confirm/reject/override.
Severity score = VAD score × Gemma confidence (not binary)

How CrowdGuard Works

A 7-stage compute-gated pipeline. Each stage only fires if the previous one says "worth looking at."

1

VAD — Always-On Filter

CLIP-TSA scores every clip for anomaly likelihood (0–1). Runs continuously, cheaply filters ~90% of normal footage.

2

Gemma — Semantic Reasoning (On-Device)

Gemma confirms/rejects anomalies with natural-language reasoning. Flags person involvement. No raw footage leaves the edge.

3

YOLOv8n — Person Localization

Detects and boxes persons in confirmed clips. Only runs when person_involved = yes. Non-person anomalies skip to logging.

4

OSNet — Re-Identification

YOLO crop → 512-dim embedding vector. Person's unique "fingerprint" for cross-camera matching. ~5–10ms per crop.

5

Cross-Camera Matching

Cosine similarity check against embedding store. Threshold ~0.7 to flag same person across cameras vs new individual.

6

Forensic Logging

Hash-chained, append-only JSON log. VAD score, Gemma reasoning, YOLO boxes, match confidence — every decision traceable.

7

Alert Dashboard

Severity score (not binary), human-in-the-loop override, explainability panel with Gemma reasoning, comparison mode.

Why CrowdGuard is Different

Every design choice directly addresses a real failure in current surveillance systems.

🔒

On-Device Privacy

Gemma runs entirely on-edge. No raw footage ever leaves the device — resolving cloud latency and privacy risk simultaneously.

🧠

Reasoning, Not Pattern Matching

Gemma provides natural-language explanations for every alert. Operators see why something was flagged — not just that it was.

📋

Forensic-Grade Audit Trail

Hash-chained, append-only logs. Every alert has a complete, tamper-evident reasoning chain that can be reconstructed and defended.

⚡

90% Alarm Reduction

Compute-gated pipeline filters noise before reasoning. Operators only see genuinely confirmed anomalies — not every motion trigger.

📹

Cross-Camera Tracking

OSNet Re-ID links the same person across camera feeds using embedding similarity. No person is tracked until semantically confirmed.

🎛️

Human-in-the-Loop

Severity scores (not binary alerts), operator override, and explainability panels. Designed for deployment reality, not just demo.

Built With

All models pretrained, zero training needed for demo. Designed for edge deployment.
